功能 5：捏云朵互动
--------------------------------

新增：
- src/components/CloudKneadInteraction.jsx
- 云朵素材建议放在 src/assets/clouds/ 或 public/assets/clouds/，按项目现有资产规范决定。
- 可选配置写入 src/config/microInteractionConfig.js

适用情绪：
- anxious
- calm

交互目标：
- anxious：揉散乱云。
- calm：把云朵捏成一颗轻轻的星。

文案：
- anxious 引导：“把乱云慢慢揉散。”
- calm 引导：“把云朵捏成一颗轻轻的星。”
- anxious 完成：“乱云散开了一点，星光进来了。”
- calm 完成：“一颗安静的小星星被你捏出来了。”

视觉重点：
不要只用几个清晰圆形拼成云朵，因为会显得廉价或像儿童贴纸。
优先采用“imagegen 生成云朵图片素材 + 轻量 CSS/SVG 动效”的混合方案。

如果环境允许 imagegen，请生成 3 张透明背景云朵素材：

1. cloud_idle.png
描述：
柔和、梦幻、半透明、星空蓝紫色调的云朵，透明背景，童话感，治愈感，适合放在深蓝星空背景上。不要写实厚重云，不要卡通粗描边，不要天气 App 图标感。云朵边缘有轻微柔光，像情绪空间里的柔光云。

2. cloud_squeezed.png
描述：
同一风格的云朵，被轻轻捏扁一点，中心微微发光，仍然柔和、半透明、星空蓝紫色调，透明背景。不要夸张变形，不要搞笑表情，不要粗描边。

3. cloud_dissolve.png
描述：
同一风格的云朵，边缘正在散成星尘，云朵变薄、变亮、即将消散，透明背景，周围有少量细小星点。不要烟雾爆炸感，不要强烈粒子爆炸，只要温柔消散。

素材要求：
- 透明背景 PNG。
- 尺寸建议 512x512 或 768x768，按项目资源大小控制。
- 文件体积尽量小。
- 风格统一。
- 如果 imagegen 只能输出非透明背景，请用可行方式去背或改为 CSS/SVG fallback。
- 不要提交超大图片。

如果 imagegen 不可用：
- 使用 fallback SVG/blurred blob 云朵。
- 必须使用 blur、radial-gradient、opacity、filter，让它不要像几个圆形拼接。
- blob 边缘要柔和，整体像云团。
- 不要用清晰圆形直接拼贴。

CloudKneadInteraction 功能要求：
- 使用云朵图片作为主要视觉。
- 支持 pointerdown / pointermove / pointerup。
- 移动端可触摸。
- 用户拖拽时：
  - 云朵轻微 scaleX / scaleY / rotate / translate。
  - idle 状态逐渐过渡到 squeezed 状态。
  - 周围出现少量星尘。
- 用户完成 3 次有效揉捏后：
  - 切换到 dissolve 状态。
  - 叠加 12 到 18 个 CSS 星尘粒子。
  - 星尘淡出后触发 onComplete。
- 有长按 fallback：长按 1.2 秒也算一次揉捏。
- 不影响页面滚动和主按钮点击。
- pointer capture 处理要稳。
- pointer-events 不要误伤其他层。

props 建议：
- emotion
- active
- onComplete
- onCancel
- reducedMotion
- variant

内部状态建议：

- 动效克制，不遮挡主流程。
- prefers-reduced-motion 下减少粒子数量和位移。

视觉验收
--------------------------------

如果可以启动本地页面，请检查以下尺寸：
- 1366x768
- 1440x900
- 390x844

如果可以使用 vision，请对截图进行检查，重点看：

1. 左下角语录：
- 是否遮挡主按钮。
- 是否遮挡纸团。
- 是否遮挡星星详情入口。
- 移动端是否过宽。

2. 合掌许愿：
- 柔光是否过强。
- 愿望星是否明显但不刺眼。
- 完成提示是否能读清。
- 是否影响主流程点击。

3. 流星雨：
- 是否像流星，不像激光。
- 是否过密。
- 是否遮挡星座或弹窗。
- 动效是否太快。

4. 开心烟花：
- 是否有庆祝感。
- 是否过亮。
- 是否遮挡星星点击。
- 是否像星花而不是爆炸特效。

5. 捏云朵：
- 云朵是否第一眼像柔和云团。
- 是否不像几个圆形拼接。
- 拖拽时是否有轻轻揉动的感觉。
- 消散时是否温柔。
- 移动端是否能正常触摸。

6. reduced-motion：
- 动效是否明显减少。
- 页面是否仍然有反馈。

如果视觉不通过，请微调 CSS、粒子数量、透明度、尺寸、位置。
