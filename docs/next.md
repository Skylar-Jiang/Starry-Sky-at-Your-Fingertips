0. 项目目标

对当前“指尖星空”页面做一次 视觉与交互重构修复，目标不是简单调样式，而是解决以下核心问题：

A. 场景切换问题

当前“海浪 / 篝火 / 云层 / 雨夜”等场景切换，本质上还是在“贴背景图”，没有真正做到：

人物与环境一起切换
人物真实地坐在海浪 / 云层 / 篝火场景中
场景元素（海浪、云层、篝火等）真实可见
角色风格保持稳定，不被改坏
B. 观测星空问题

当前“观测星空”存在：

星座连线逻辑错误
生成的是任意弯曲轨迹，而不是预设星座
点击前后星星位置不一致
背景出现横向分层 / 拼接线
微光效果错误，被画成断层、折线或异常结构
没有真正可识别的星座轮廓
1. 最终效果目标

最终页面必须满足以下目标：

1.1 场景切换目标

切换不同环境时，不是替换整张背景图，而是替换 场景元素层：

角色层：小王子 / 小狐狸 / 花朵
承载物层：海浪 / 云层 / 篝火地面 / 雨夜地面
环境氛围层：波光 / 火光 / 云边光 / 雨滴 / 月光
基础星空层：统一稳定，不随切换乱变
1.2 角色目标
保持当前小王子、小狐狸、花朵的气质、风格、识别度
不能把小王子改坏、改脸、改气质，如果图片本身有缺陷 用imagegen重新生成风格一致的图片
不能把狐狸和花朵画崩
切换场景时，角色要进入该场景，而不是固定在页面底部
1.3 观测星空目标
使用 预设星座模板
至少支持一个简单、用户一眼能识别的星座，例如 北斗七星
使用固定星点位置与固定连线关系
点击前后基础星空一致
没有横向拼接线 / 分层线
微光是柔和小光点，而不是路径、折线、几何断裂
2. 问题拆解
2.1 当前场景切换问题拆解
问题 1：环境被当作背景板

当前做法大概率是：

页面上有一个固定主场景
切换环境时，只替换某个背景图层
左下角角色仍然保持原位置 / 原结构
正确修复方向

改为真正的分层场景系统：

baseSkyLayer
sceneEnvironmentLayer
sceneCharacterLayer
sceneEffectLayer
constellationLayer
uiLayer

切换环境时要替换的是：

sceneEnvironmentLayer
sceneCharacterLayer
sceneEffectLayer

而不是只换一张图。

2.2 当前观测星空问题拆解
问题 2：星座被错误理解成“路径”

现在生成的是：

一条贝塞尔曲线 / 拖拽轨迹
星点只是散落在旁边
用户看不出这是任何星座
正确修复方向

必须改成：

预设星座模板
固定的主星位置
固定的连线顺序
连线是直线段 / 极轻微平滑折线，而不是随意曲线

例如北斗七星：

7 颗主星
明确的“勺身 + 勺柄”结构
用户一眼能认出形状
2.3 当前背景分层问题拆解
问题 3：横向明显断层 / 分层

原因可能包括：

多段背景图拼接
渐变遮罩边界可见
叠加层高度不一致
blend / mask / canvas 图层未对齐
观测状态和普通状态用了两套背景
正确修复方向
基础星空只保留 单一来源
普通态与观测态使用 同一套基础星空
观测态只在基础星空上叠加 constellationLayer
所有大面积背景层必须避免出现边界线
3. 解决方案总结构
3.1 页面视觉架构

建议最终页面结构：

<AppScene>
  <BaseSkyLayer />
  <SceneEnvironmentLayer />   // 海浪 / 云层 / 篝火 / 雨夜
  <SceneCharacterLayer />     // 小王子 / 狐狸 / 花朵
  <SceneEffectLayer />        // 火光 / 波光 / 雨滴 / 云边柔光
  <ConstellationLayer />      // 观测星空专用
  <UILayer />                 // 标题、按钮、筛选器等
</AppScene>
3.2 场景系统设计

每个场景应有独立配置，例如：

type SceneKey = 'ocean' | 'campfire' | 'cloud' | 'rain';

interface SceneDefinition {
  key: SceneKey;
  name: string;
  environmentAsset: string;
  characterLayout: {
    prince: { x: number; y: number; scale: number };
    fox?: { x: number; y: number; scale: number };
    rose?: { x: number; y: number; scale: number };
  };
  effects: string[];
}
3.3 星座系统设计
type ConstellationKey = 'bigDipper';

interface ConstellationDefinition {
  key: ConstellationKey;
  stars: { id: string; x: number; y: number; type: 'main' | 'accent' }[];
  edges: [string, string][];
}

5. 执行步骤（推荐顺序）
Step 1：先冻结基础星空

目标：

确定一张唯一的基础星空
普通态与观测态共用这张星空
不允许切换状态后换另一套星点位置

验收：

点击“观测星空”前后，星点分布一致
不再出现横向拼接线
Step 2：重建场景图层系统

目标：

从“整图背景替换”改为“多图层组合”
搭建环境层、角色层、特效层

验收：

切换环境时，角色和环境一起变
人物不再像贴在页面底边
Step 3：生成 / 重做场景素材

目标：

用 imagegen 生成海浪、篝火、云层、雨夜承载面
用 imagegen 保持原风格重做角色层素材

验收：

海浪场景看得到水
篝火场景看得到火
云层场景看得到云
角色没有画崩
Step 4：接入素材并调布局

目标：

让人物真实“坐在”环境承载物上
每个场景有独立布局

验收：

海浪：人物坐在海边 / 水边平台
篝火：人物与狐狸围绕火堆
云层：人物坐在云层上
雨夜：人物处于雨夜地面环境
Step 5：重建观测星空

目标：

使用固定星座模板
先实现一个最简单的“北斗七星”
固定主星位置、固定连线关系

验收：

观测态能明显看出北斗七星轮廓
不再出现随意大弯曲线
连线细、连续、清晰
Step 6：加回微光交互

目标：

微光只作为柔和的点状装饰和收集反馈
不得干扰星座主结构

验收：

微光是小发光点
收集时回到主星附近
不形成大折线 / 拼接结构
Step 7：vision 截图验证

至少验证以下页面状态：

默认页面
海浪场景
篝火场景
云层场景
雨夜场景
观测星空未开启
观测星空开启（北斗七星显示）

每个状态都要检查：

角色是否正确
环境是否真实可见
是否有拼接线
星星位置是否稳定
星座是否可识别
