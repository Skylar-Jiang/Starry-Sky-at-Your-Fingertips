第一阶段统一技术栈约定
1. 前端框架

统一使用：

React + Vite

原因：第一阶段主要是前端 Demo，任务书里也已经按 App.jsx、components/、config/、utils/ 这种 React 项目结构来规划。

2. 编程语言

统一使用：

JavaScript + JSX

也就是文件使用：

App.jsx
MainScene.jsx
DiaryModal.jsx
PaperNote.jsx
PaperBall.jsx
StarLayer.jsx

第一阶段不建议使用 TypeScript，除非 A 和 B 都很熟。
因为你们现在最重要的是快速跑通 Demo，TypeScript 会多一层类型维护成本。

但是接口文档里可以保留 type 写法，作为说明，不一定真的创建 .ts 文件。

3. 样式方案

统一使用：

普通 CSS

建议：

src/index.css
src/App.css

或者每个组件一个 CSS 文件也可以：

DiaryModal.css
PaperBall.css
SceneEffects.css

但不要 A 用 Tailwind，B 用 CSS Modules，C 又写内联样式。第一阶段统一普通 CSS 最稳。

4. 包管理工具

统一使用：

npm

运行方式：

npm install
npm run dev

不要一个人用 pnpm，另一个人用 yarn，否则锁文件容易冲突。

5. 动画实现方式

第一阶段统一优先使用：

CSS transition / CSS animation

不强制装动画库。

也就是说：

transition: all 0.6s ease;
animation: twinkle 2s infinite;

如果 B 想用 framer-motion，必须提前告诉 A，并且两边统一安装：

npm install framer-motion

但我建议第一阶段先不用，避免额外依赖。

6. 数据存储

统一使用：

localStorage

技术 A 负责封装：

src/utils/storage.js

B 不直接操作 localStorage。
B 只负责把投掷后的星星坐标回传给 A。

7. 第一阶段不接后端

不做：

后端服务
数据库
登录系统
云端存储
API Server

所有数据都在前端状态和 localStorage 里。

注意：

A 创建 record。
B 接收 record。
B 投掷完成后只回传 star。
A 负责把 star 写回 record。
5. A 给 B 的 props 统一

A 在 MainScene 或父组件里传给 B：

<PaperNote
  record={pendingRecord}
  onFoldComplete={handleFoldComplete}
/>

<PaperBall
  record={pendingRecord}
  onThrowComplete={handleThrowComplete}
/>

<StarLayer
  records={records}
  onSelectStar={handleSelectStar}
/>

<SceneEffects
  emotion={currentEmotion}
/>

如果你们想更简单，也可以让 B 做一个总组件：

<PaperFlow
  record={pendingRecord}
  onThrowComplete={handleThrowComplete}
/>

这样 A 只需要对接一个组件。

6. B 给 A 的回调统一

B 投掷完成后，必须调用：

onThrowComplete({
  recordId: record.id,
  star: {
    id: "star_001",
    x: 620,
    y: 160
  }
});

A 接收后更新 records。

第一阶段统一技术栈：

1. 使用 React + Vite。

14. 点击星星时，B 调用 onSelectStar(record)，A 打开详情弹窗。
